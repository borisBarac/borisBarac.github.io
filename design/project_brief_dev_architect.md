# Project Brief: DEV_ARCHITECT Portfolio

## 1. Project Overview
A high-fidelity personal portfolio designed for senior software architects and developers. The project, titled **DEV_ARCHITECT**, emphasizes technical rigor, systems thinking, and clean, modular design.

## 2. Design Vision
- **Theme Name:** Technical Precision
- **Aesthetic:** High-contrast dark mode with terminal-inspired accents.
- **Visual Language:** Uses a grid-based layout and monospace typography to evoke a code environment while maintaining professional readability.
- **Primary Color:** `#00ff41` (Terminal Green)
- **Surface Color:** `#131313` (Deep Charcoal)

## 3. Core Screens & Functionality
### Home Page (`SYSTEM.STATUS: ONLINE`)
- **Hero Section:** Bold headline "ENGINEERING ARCHITECTURES FOR THE WEB" with a technical grid background.
- **Directory Index:** High-level navigation to Projects and Career Log.
- **System Dependencies:** A visual "config file" representation of the tech stack (TypeScript, Go, React, Kubernetes, etc.).

### Projects Gallery (`PROJECTS_`)
- **Featured Builds:** Showcases complex implementations like a "Distributed Task Scheduler" and "WASM Vector DB".
- **Technical Metadata:** Each project lists core tech, infrastructure, and stack bindings.
- **Interactions:** Direct links to "VIEW_DEMO" and "SOURCE_CODE".

### About & Experience (`EXPERIENCE`)
- **Professional Timeline:** Detailed career history with skill tags (Rust, Kubernetes, AWS).
- **Education & Certs:** Structured list of academic credentials and active certifications.
- **WhoAmI Section:** Narrative on systems design philosophy and personal interests.

### Contact (`INITIATE CONTACT.`)
- **Terminal Form:** A minimalist, shell-inspired interface for inquiries.
- **Direct Links:** Quick access to GitHub, LinkedIn, and Twitter.

## 4. Technical Specifications
- **Frameworks:** React / Next.js
- **Styling:** Tailwind CSS with a custom design system integration.
- **Responsiveness:** Desktop-first high-density layout.
- **Typography:** Hanken Grotesk for headings; monospace for technical data.

## 5. Future Roadmap
- Implementation of a real-time terminal simulation for the contact form.
- Dynamic project filtering by "Stack" or "Infrastructure".
- Integration with GitHub API for live repository stats.
